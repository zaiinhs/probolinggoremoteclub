# PRD Website Probolinggo Remote Club

## Informasi Dokumen

- **Produk**: Website Probolinggo Remote Club
- **Versi**: 1.0
- **Status**: Draft awal
- **Tujuan utama**: Menjadi website resmi komunitas untuk memperkenalkan komunitas, mempublikasikan kegiatan, mengelola pendaftaran anggota, dan menyediakan pusat informasi komunitas.

## Ringkasan Produk

Website Probolinggo Remote Club akan menjadi hub digital utama bagi komunitas pekerja remote di wilayah Probolinggo. Website ini berfungsi sebagai media informasi, kanal pendaftaran anggota, publikasi agenda kegiatan, dokumentasi aktivitas komunitas, dan sarana untuk menghubungkan pekerja remote dengan komunitas lokal.

Website ditujukan untuk komunitas yang bersifat non-profit, gratis, inklusif, dan terbuka bagi seluruh pekerja remote lintas profesi.

## Latar Belakang Produk

Saat ini belum ada media terpusat yang mewakili komunitas pekerja remote Probolinggo secara resmi. Informasi komunitas berisiko tersebar di berbagai kanal dan sulit diakses oleh calon anggota baru. Karena itu, website dibutuhkan sebagai pusat identitas, informasi, dan operasional dasar komunitas.

## Tujuan Produk

1. Memperkenalkan Probolinggo Remote Club kepada publik.
2. Memudahkan orang untuk memahami visi, misi, dan bentuk kegiatan komunitas.
3. Menyediakan alur pendaftaran anggota yang sederhana.
4. Menampilkan agenda meetup, coworking, dan kegiatan komunitas lainnya.
5. Menjadi pusat informasi resmi terkait komunitas.
6. Mendukung pengelolaan komunitas secara lebih rapi dan terdokumentasi.

## Sasaran Pengguna

### Pengguna Utama

1. Pekerja remote asal Probolinggo.
2. Pekerja hybrid yang berdomisili di Probolinggo.
3. Freelancer dari berbagai bidang.
4. Orang yang tertarik masuk ke dunia kerja remote dan ingin terhubung dengan komunitas.

### Pengguna Internal

1. Pengelola komunitas.
2. Admin website.
3. Koordinator kegiatan atau event organizer komunitas.

## Persona Pengguna

### Persona 1: Remote Worker Aktif

- Bekerja penuh secara remote.
- Ingin bertemu sesama pekerja remote di Probolinggo.
- Membutuhkan informasi event dan tempat kerja bareng.

### Persona 2: Freelancer Lintas Bidang

- Tidak bekerja di perusahaan tetap.
- Ingin memperluas koneksi dan mendapat lingkungan yang suportif.
- Tertarik mengikuti meetup santai dan coworking session.

### Persona 3: Calon Anggota Baru

- Baru mengetahui komunitas ini.
- Ingin tahu komunitas ini untuk siapa, kegiatannya apa, dan bagaimana cara bergabung.
- Membutuhkan proses join yang cepat dan jelas.

### Persona 4: Admin Komunitas

- Mengelola halaman website, anggota, event, dan konten.
- Membutuhkan dashboard sederhana untuk operasional komunitas.

## Problem Statement

1. Calon anggota belum memiliki satu tempat resmi untuk mengenal komunitas.
2. Informasi event dan kegiatan komunitas belum terpusat.
3. Admin membutuhkan sistem dasar untuk mengelola data anggota dan kegiatan.
4. Komunitas membutuhkan identitas digital yang kredibel dan mudah diakses.

## Value Proposition

Website Probolinggo Remote Club menawarkan:

1. Satu rumah digital resmi untuk komunitas pekerja remote Probolinggo.
2. Informasi komunitas yang jelas dan mudah dipahami.
3. Pendaftaran anggota yang sederhana dan gratis.
4. Kalender kegiatan komunitas yang terorganisir.
5. Ruang publikasi untuk membangun ekosistem kerja remote lokal.

## Ruang Lingkup MVP

Fokus MVP adalah membuat website komunitas yang informatif dan fungsional untuk kebutuhan operasional dasar.

### Fitur MVP Prioritas Tinggi

1. Halaman beranda.
2. Halaman tentang komunitas.
3. Halaman daftar/join komunitas.
4. Halaman event atau kegiatan.
5. Halaman daftar venue atau tempat rekomendasi untuk kerja remote.
6. Halaman artikel atau update komunitas.
7. Halaman kontak dan tautan komunitas.
8. Dashboard admin dasar untuk kelola konten dan data.

### Fitur MVP Prioritas Menengah

1. Profil anggota publik opsional.
2. RSVP atau pendaftaran event.
3. Newsletter atau subscription email.
4. Galeri dokumentasi kegiatan.

### Fitur Di Luar MVP

1. Forum komunitas penuh.
2. Direct messaging antar anggota.
3. Job board khusus remote.
4. Payment system.
5. Sistem chapter atau cabang komunitas.

## Kebutuhan Fungsional

### 1. Halaman Beranda

Website harus memiliki halaman beranda yang menampilkan:

1. Penjelasan singkat tentang Probolinggo Remote Club.
2. CTA untuk bergabung.
3. Highlight event terdekat.
4. Highlight venue atau tempat kerja remote-friendly.
5. Ringkasan manfaat bergabung.
6. Testimoni atau kutipan anggota jika tersedia.

### 2. Halaman Tentang

Website harus menyediakan informasi tentang:

1. Latar belakang komunitas.
2. Visi dan misi.
3. Nilai komunitas.
4. Siapa yang bisa bergabung.
5. Prinsip non-profit dan gratis.

### 3. Pendaftaran Anggota

Website harus memiliki form pendaftaran anggota dengan data minimal:

1. Nama lengkap.
2. Email.
3. Nomor WhatsApp opsional.
4. Domisili atau kecamatan.
5. Pekerjaan atau profesi.
6. Status kerja remote atau hybrid.
7. Alasan bergabung opsional.

Setelah submit:

1. Data tersimpan di sistem.
2. Pengguna melihat pesan sukses.
3. Admin dapat melihat data pendaftar.

### 4. Event atau Kegiatan

Website harus bisa:

1. Menampilkan daftar event mendatang dan event lampau.
2. Menampilkan detail event.
3. Menampilkan tanggal, waktu, lokasi, kapasitas, dan deskripsi event.
4. Mengizinkan pengguna melakukan RSVP atau pendaftaran event.

### 5. Venue Remote-Friendly

Website harus menyediakan daftar tempat yang cocok untuk kerja remote di Probolinggo, dengan data seperti:

1. Nama tempat.
2. Lokasi.
3. Deskripsi singkat.
4. Fasilitas seperti Wi-Fi, colokan, indoor, outdoor, makanan, parkir.
5. Jam operasional opsional.
6. Apakah cocok untuk coworking komunitas.

### 6. Artikel atau Update

Website harus dapat mempublikasikan:

1. Artikel komunitas.
2. Pengumuman.
3. Dokumentasi kegiatan.
4. Tips seputar kerja remote atau ekosistem lokal.

### 7. Kontak dan Tautan Komunitas

Website harus memuat:

1. Email komunitas.
2. Link WhatsApp group atau formulir masuk grup, bila ada.
3. Link media sosial.
4. Informasi kontak admin.

### 8. Dashboard Admin

Admin minimal harus dapat:

1. Login ke sistem.
2. Mengelola halaman statis.
3. Mengelola event.
4. Mengelola venue.
5. Mengelola artikel.
6. Melihat daftar anggota.
7. Melihat pendaftar event.

## Kebutuhan Non-Fungsional

1. Website harus mobile-friendly.
2. Website harus cepat diakses pada koneksi internet umum.
3. Website harus mudah dikelola oleh admin non-teknis.
4. Website harus memiliki struktur SEO dasar.
5. Website harus aman untuk data anggota dasar.
6. Website harus mudah dikembangkan untuk fitur lanjutan di masa depan.

## User Flow Utama

### Flow 1: Calon Anggota Bergabung

1. Pengguna membuka beranda.
2. Pengguna membaca penjelasan komunitas.
3. Pengguna klik tombol `Gabung Komunitas`.
4. Pengguna mengisi form pendaftaran.
5. Sistem menyimpan data.
6. Pengguna menerima konfirmasi bahwa pendaftaran berhasil.

### Flow 2: Pengguna Mencari Event

1. Pengguna membuka halaman event.
2. Pengguna melihat daftar event mendatang.
3. Pengguna membuka detail event.
4. Pengguna memilih ikut dan mengisi RSVP jika fitur tersedia.

### Flow 3: Admin Membuat Event Baru

1. Admin login.
2. Admin membuka dashboard.
3. Admin membuat event baru.
4. Event tampil di halaman publik.

1. Komunitas masih berada pada tahap awal sehingga kebutuhan sistem dibuat sederhana.
2. Keanggotaan komunitas gratis.
3. Event dapat berupa kegiatan gratis dan sederhana seperti meetup atau coworking bersama.
4. Admin internal komunitas akan mengelola konten secara manual.

## KPI Awal

### KPI Komunitas

1. Jumlah pendaftar anggota.
2. Jumlah anggota aktif yang hadir ke event.
3. Jumlah event yang berhasil dipublikasikan dan dijalankan.
4. Jumlah venue lokal yang terdaftar.

### KPI Website

1. Jumlah kunjungan bulanan.
2. Conversion rate dari pengunjung menjadi pendaftar.
3. Jumlah RSVP event melalui website.
4. Waktu rata-rata di halaman utama dan halaman event.

## Risiko dan Catatan

1. Jika proses administrasi terlalu rumit, calon anggota bisa batal bergabung.
2. Jika admin dashboard terlalu kompleks, pengelolaan website bisa tersendat.
3. Jika konten tidak rutin diperbarui, website bisa terlihat tidak aktif.
4. Jika data venue tidak akurat, pengalaman pengguna bisa menurun.

## Definisi Sukses MVP

MVP dianggap berhasil bila:

1. Website resmi dapat diakses publik.
2. Calon anggota dapat mendaftar dengan mudah.
3. Admin dapat mempublikasikan event dan konten tanpa hambatan besar.
4. Website menjadi pusat informasi resmi komunitas yang konsisten dipakai.
